В логгер добавлен функционал по перезагрузке параметров во время выполнения приложения.
Возможна периодическая проверка времени изменения файла параметров и его перезагрузка
при необходимости. За это отвечает парамерт configReloadInterval в файле параметров логгера.
Значение этого параметра задается в секундах. Если параметр не задан или его значение равно нулю,
периодическая проверка и перезагрузка не осуществляется.
Добавлен метод Logger::Reload(), который перзагружает файл параметров и может быть вызван
в любой момент выполнения программы, например из обработчика сигнала.
Добавлен метод Logger::Store(), который позволяет сохранить уровни логгирования категорий,
измененных во время исполнения программы. 

в логгере, наряду с dayfile аппендером, появился intervalfile аппендер, позволяющий 
прокручивать логи через любой заданный интервал времени.

В файле logger.properties можно задать следующие параметры:
suffixFormat - этот параметр позовляет задать суффикс файла с логами, порядок масок:
год, месяц, день, час, минуты, секунды. порядок не может быть изменен.
по-умолчанию:
suffixFormat=%04d%02d%02d%02d%02d%02d (smsc.log.20071015161428)
пример:
appender.aFile1.intervalfile.suffixFormat=%04d-%02d-%02d %02d:%02d:%02d (smsc.log.2007-10-15 16:14:28)

interval - длительность интервала в формате hh:mm:ss.
по-умолчанию:
interval=24:00:00
пример:
appender.aFile1.intervalfile.interval=01:01:15

maxindex - количество интервалов за которые хранить логи. если значения равно нулю не удалять файлы логов.
по-умолчанию:
maxindex=0
пример:
appender.aFile1.intervalfile.maxindex=10

postfix - это параметр позволяет задать постфикс для прокрученных файлов с логами, т.е. файлов в которые больше не пишутся логи
по-умолчанию:
postfix=.log
пример:
appender.aFile1.intervalfile.postfix=.old (smsc.log.2007-10-15.old)
